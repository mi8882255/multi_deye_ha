# Deye Multi-Inverter Reader — Home Assistant Addon

## Описание

Аддон читает данные с инверторов Deye по Modbus TCP / Solarman V5 и публикует в MQTT с автоматическим обнаружением устройств в Home Assistant.

## Поддерживаемые модели

- `deye-sun-3phase-lv` — SUN-6K/8K/10K/12K (3-фазный сетевой)
- `deye-sun-3phase-hv` — SUN-15K/20K (3-фазный сетевой HV)
- `deye-sg05lp3` — SUN-5K÷20K-SG05LP3-EU (3-фазный гибрид)
- `deye-hybrid-1p` — SUN-5K/6K-SG03LP1/SG04LP1-EU (1-фазный гибрид)
- `deye-sg02lp1` — SUN-8K÷16K-SG02LP1/SG01LP1-EU (1-фазный гибрид, 3 MPPT)
- `deye-micro` — SUN-300G3÷2000G3, SUN-*K-G (микроинвертор/строковый)

## Конфигурация

### Инвертор

| Поле | Обязательно | По умолчанию | Описание |
|------|-------------|--------------|----------|
| `id` | да | — | Уникальный идентификатор |
| `host` | да | — | IP-адрес инвертора/логгера |
| `port` | нет | 502 | Порт (502 = Modbus TCP, 8899 = Solarman V5) |
| `unitId` | нет | 1 | Modbus Unit ID (1–247) |
| `model` | да | — | Модель инвертора |
| `name` | нет | = id | Отображаемое имя |
| `serialNumber` | нет | — | Серийный номер логгера (для Solarman V5) |
| `retries` | нет | 4 | Количество повторных попыток |
| `retryMinDelay` | нет | 200 | Мин. задержка retry (мс) |
| `retryMaxDelay` | нет | 1000 | Макс. задержка retry (мс) |

### Общие настройки

| Поле | По умолчанию | Описание |
|------|--------------|----------|
| `read_interval` | 10 | Интервал чтения (сек) |
| `report_interval` | 30 | Интервал отчёта в MQTT (сек) |
| `gap_tolerance` | 10 | Объединение регистров с разрывом до N адресов |
| `stale_threshold_seconds` | 120 | Порог устаревания кеша (сек) |
| `slow_poll_multiplier` | 6 | Медленные сенсоры — каждый N-й цикл |

## Сосуществование с облаком Solarman

Аддон поддерживает параллельную работу с облаком Solarman. Соединение открывается только на время чтения и сразу закрывается. При конфликте — автоматический retry с случайной задержкой.

## MQTT

Состояние публикуется в `<topic_prefix>/<inverter_id>/state` как JSON.
Доступность — `<topic_prefix>/status` (`online`/`offline`).
Discovery-топики создаются автоматически при старте.
